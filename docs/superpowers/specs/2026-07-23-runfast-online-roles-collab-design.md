# 跑得快记分 联机重构：角色认领 + 协作式记分 设计文档

日期：2026-07-23
状态：已与用户确认
前置：局域网自建服务器见 `2026-07-23-runfast-lan-online-design.md`（本方案在其上扩展**联机交互模型**，后端仍是局域网 `server.js`，不改变"手机同 WiFi 扫码进"的部署）。

## 1. 背景与目标

现联机模型：房主预先输入所有玩家名字 → 建房即"进行中" → **一人记分、其他人被记分锁挡住或只观战**。
新模型：**按座位认领角色 → 每人只填自己那格、实时同屏协作记分**，更贴近真人围桌各按各的手机。

目标：
1. 房主建房设**人数 + 各座名字**；建房后进入**等待**，各人（含房主）**认领座位（名字）**。
2. **一台手机可认领多座**（替没带手机的人）；**满员才能开局**；满员后迟到者**观战**。
3. **记一局改协作**：赢家先定，各座各填自己那格，实时同屏，全部填完后**谁都能点保存**。
4. 中途房主可**加人 / 标离场**；**改/删某一局仍房主单人重填**。
5. 顶部显示 **「N 人在玩 · M 人观战」**。
6. 移除旧的 **「允许他人修改」开关** 与 **记分锁**（被协作模型取代）。
7. **本地单机模式完全不变**。

## 2. 术语与角色

- **阶段 `phase`**：`'lobby'`（等待认领）| `'playing'`（对局中）| `'finished'`（已结束）。
- **房主 `creatorUid`**：建房设备。可管理座位 / 开局 / 结束 / 改删局 / 关房；**可越权代填任意座**。
- **座位 seat**：`{ name, claimedBy }`。`claimedBy` = 认领该座的 `deviceId`，未认领为 `null`。
- **观战者**：已连接但**未认领任何座位**的设备。只读。

## 3. 房间生命周期

建房(lobby) → 各人认领 → 满员 → 房主「开始记分」(playing) → 记若干局 → 房主「结束本场」(finished) → 各端存本地历史 → 房主「关闭房间」。

- **lobby**：显示座位表（灰=未认领，高亮=已认领）。房主「开始记分」**仅当每座都已认领**才可点。
- **playing**：协作记分（§5）。
- **finished**：结算页；房主出现「关闭房间」（复用现有逻辑）。

## 4. 座位与认领

- **认领**：点一个**灰色座位** → 写 `seats[i].claimedBy = 本机 deviceId` → 变高亮。已认领的座位不可再被别人点（除非空出）。
- **代填多座**：一台手机可认领**多个座位**（给没带手机的人）。认领动作可对多个空座重复。记分时，一台手机负责它认领的**所有**座位。
- **满员判据**：所有座位 `claimedBy != null`。因可代填多座，满员始终可达成。
- **抢座并发**：两人同时点同一空座 → 服务器**条件写（CAS：仅当该座为空才写）**；后到者被拒并经广播看到已被占，提示"这座已被认领，换一个"。
- MVP **不做**主动换座/让座；换人靠房主「标离场」腾座后重新认领（§6）。

## 5. 协作式记一局（核心改动）

对局中，牌桌上有一张**大家共享的草稿 `draft`**：

```
draft = { winner: <name>|null, entries: { <loserName>: { cardsLeft, shutout } } }
```

流程：
1. **赢家先定**：任一在场玩家（通常赢家本人）点某座 = `winner`。赢家该局固定 0 张、不填。保存前可改 winner（改 winner 时：清掉**新** winner 的 entry —— 赢家不填；把**旧** winner 纳入需填名单）。
2. **各填各的**：其余每个**在场**座位，由**认领它的手机**（或**房主**）填 `cardsLeft`(0–10) / 全关。写 `draft.entries[name]`。**非本座、非房主不能写该格**。
3. **实时同屏**：`draft` 每次变更广播；**所有在场端（含观战）都看到最新填写进度**——不再锁人、不再"只有一个人能看"。
4. **确认保存**：当 `winner` 已定 **且** 每个在场输家的 `entries` 都已填 → 「确认保存」亮；**任一在场玩家（非观战）可点** → 把 `draft` 转成一局 append 到 `session.rounds`、清空 `draft`、进入下一局。**保存幂等**（草稿已被清空则忽略重复点，防双存）。
- **观战者**：只看，不能填、不能点保存、不能定赢家。
- **离场玩家**：不参与当局 draft。

## 6. 中途管理 / 改删局 / 迟到 / 断线

- **玩家管理（仅房主）**：
  - **加人**：追加一个座位（name），lobby/playing 皆可；新座灰色待认领，不影响历史局（该名从加入起参与新局）。
  - **标离场**：把某座标离场 → 移出后续对局（`session.activePlayers` 去掉该名），**历史分保留**；其 `claimedBy` 释放（座位空出，可被新人认领或本人回归）。
- **改 / 删某一局（仅房主）**：房主**单人**重填 / 删除该局（复用现有改删局 UI，不走协作）。
- **迟到者（满员后进来）**：无空座 → **观战**；等有人离场腾座 → 可认领入座。
- **断线**：座位**不因断线释放**（息屏 / 短断网仍算在玩）；回来自动接最新。真走人由房主标离场。

## 7. 在玩 / 观战人数（presence，临时态）

- **在玩人数** = 当前活跃座位数（`claimedBy != null` 且未离场）。纯房间状态，本地可算。
- **观战人数** = 已连接但未占任何座位的设备数。
- **实现**（不落地、不进房间对象）：SSE 连接带 deviceId（`/rooms/:code/events?dev=<id>`）；服务器按房维护**在线 deviceId 集合**，连接/断开即变；集合变化时向该房推一帧 `event: presence\ndata: {devices:[<deviceId>...]}`。客户端：`观战 = devices 中未出现在任何 seats.claimedBy 的数量`。
- 显示：顶部状态条 **「N 人在玩 · M 人观战」**（lobby 显示已认领/总座位，如"已就座 2/4 · 观战 1"）。

## 8. 数据模型

房间（**仅联机**）：
```
rooms/<code>: {
  creatorUid,
  phase,                                  // 'lobby' | 'playing' | 'finished'
  updatedAt,
  seats: [ { name, claimedBy } ],         // 与 session.players 同名同序；claimedBy=deviceId|null
  draft: null | { winner: <name>|null, entries: { <name>: { cardsLeft, shutout } } },
  session: { ...与本地 session 同构（id/createdAt/pricePerCardFen/players/activePlayers/rounds/status）... }
}
```
- `seats[i].name === session.players[i]`。离场：名字留在 `players`、移出 `activePlayers`、该座 `claimedBy` 置空。
- **presence 不在房间对象内**（临时态，不落地 `server-data.json`）。
- `session` 同构 ⇒ `logic.js` 结算 / 积分 / 战绩全部复用，零改动。

## 9. 并发与一致性

- 核心：多人同时写各自 `entries` **不能整房覆盖丢更新**。
- 后端加**字段级写**：`PATCH /rooms/:code`（体 `{path, value}`，服务器按路径 set 后广播），或专用路径写。`sync.js` 的 `applyEvent` 本就按路径设计，天然复用；整房 `PUT` 保留给房主管理/改局。
- 认领座位、定 winner、保存成局也走字段级写：
  - **抢座 CAS**：`seats[i].claimedBy` 仅当当前为 `null` 才允许写为自己；否则冲突。
  - **保存原子**：`draft → round` 的转换在服务器端原子完成（或客户端 CAS：仅当 `draft` 非空才提交），防双存。

## 10. 权限（服务器强制，deviceId 为身份）

- **认领空座**：任何在房设备可把**空座**认领给自己。
- **写 `draft.entries[name]`**：仅该座 `claimedBy === me` 或 `me === creator`。
- **定 winner / 清 draft / 保存成局**：任一**持座设备**（某 `seats.claimedBy === me`）或 `creator`。观战者（无座）不可写。
- **phase 切换（开局/结束）、加人/标离场、改删局、关房**：仅 `creator`。
- 玩家名 `validName`、`esc()` 渲染、导入校验等安全项不变。

## 11. 与现有代码的关系

- **server.js**：新增字段级写（PATCH/路径写）+ 抢座/保存 CAS + presence 跟踪与 `event: presence`。保留整房 PUT/DELETE 给房主管理与改局。
- **sync.js**：`subscribe` 带 deviceId；新增 patch/claim/定winner/保存 等窄接口 + `onPresence` 回调；`applyEvent` 复用；**移除记分锁 `activeLock`**（被协作取代）。
- **app.js**：联机流程重构——建房后进 lobby；座位认领 UI；协作记分 UI（替换单人记一局流程与记分锁）；顶部人数；玩家管理适配座位；**移除「允许他人修改」开关**。**本地单机路径不变。**
- **logic.js / share-card.js / build.js**：不变。
- **移除**：`allowEdit` 开关、`editing` 记分锁及其 e2e 断言。

## 12. 测试策略

- **纯函数**（`node --test`）：座位派生 `activePlayers`、`draft → round` 转换、presence 计数（在线集合 − 持座）、满员判据、保存可用判据（winner 定 + 各输家填齐）。
- **server**：字段级写不丢更新、抢座 CAS 冲突、保存幂等、presence 事件、权限（非持座写被拒、非房主管理被拒）。
- **e2e（多标签，各设 deviceId）**：认领→满员→开局；协作填（两座并发填互不覆盖）；赢家改；保存幂等；观战只读；加人/离场；改删局；人数显示；断线重连接最新。

## 13. 范围外

- 主动换座 / 让座 UI（MVP 用"离场 + 重认领"替代）。
- 断线自动让座、座位超时回收。
- 语音 / 聊天 / 表情 / 旁观人数上限。
- 异地联机（仍局域网；将来云迁移沿用本同构模型）。
