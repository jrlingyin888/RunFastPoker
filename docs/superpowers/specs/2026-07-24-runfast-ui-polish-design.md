# 记分页界面精简设计（更多菜单 / 原生分享 / 明细独立页 / 我标识）

**日期：** 2026-07-24
**背景：** 联机二期上线后，记分主页把「比分 + 草稿 + 4 个管理按钮 + 全部局列表」堆在一屏，手机上过于拥挤；邀请只会复制文本；排行榜里认不出自己。

**目标：** 在不改动权限模型与协作记分交互的前提下，精简记分页布局、把管理项收进底部菜单、邀请改为原生分享优先、局明细独立成页、比分行标出「我」。

---

## 已确认的决策

| 议题 | 决定 |
|---|---|
| 分享方式 | **原生优先 + 自建面板降级**（局域网 HTTP 非安全上下文下 `navigator.share` 不存在） |
| 菜单归属 | ⋯ 菜单装管理项；**「每局明细」入口留在正文** |
| 「我」语义 | **我持有的每个座位都标**（房主代占多座则多行都标） |
| 改造范围 | **本地单机模式同步改造**（结构一致、组件复用） |
| 弹出形式 | **底部弹出面板**（菜单与分享面板共用一个组件） |

---

## 1. 共用组件

### 1.1 底部弹出面板 `sheet`

新增两个模块内函数（不进 `App`，不进 `view` 路由状态）：

- `closeSheet()`：移除 `#sheet` 节点（幂等）。
- `openSheet(items)`：`items = [{label, onclick, danger?}]`。创建 `#sheet.sheet-mask` 挂到 `document.body`，内部 `.sheet` 底部卡片依次渲染 `.sheet-item`，末尾固定一个「取消」项。

**关闭时机**：点遮罩本身、点任一 `.sheet-item`（含取消）。实现为容器上的一个委托监听器——按钮的内联 `onclick` 在目标阶段先执行，容器监听器随后关闭，因此「执行动作 → 关闭」顺序天然正确。

**为什么挂 body 而不是进 `view`**：联机端 `onRoom` 会频繁 `render()` 重绘 `#app`；面板挂在 `#app` 之外就不会被重绘抖掉。作为代价，需在 `go()` 内调用 `closeSheet()`，避免切页后面板残留。

### 1.2 `topbar` 增加右侧操作区

`topbar(title, backJs, actionsHtml)` 增加第三个可选参数，渲染在 `.topbar` 右侧的 `.actions` 容器（`margin-left:auto`）。`.topbar .title` 加溢出省略，避免标题长时挤掉操作区。

### 1.3 CSS 新增（`src/style.css`）

`.topbar .actions`、`.icon-btn`（顶栏图标按钮）、`.sheet-mask`、`.sheet`、`.sheet-item`（`.danger` 变体）、`.me-tag`（金色小标签）。`.sheet` 底部内边距带 `env(safe-area-inset-bottom)`。

## 2. 顶栏收纳（优化点 1 + 2）

### 布局变化

```
改前  [已记 3 局 · 1元/张]
      [● 房号 999274 · 3人在玩 · 0人观战]  [邀请][退出]
      比分卡 / 草稿区
      [玩家管理][作废本场][🔒改局仅房主]
      [结束本场]
      局列表…（很长）

改后  [已记 3 局 · 1元/张]            [分享] [⋯]
      [● 房号 999274 · 3人在玩 · 0人观战]      ← 只剩状态
      比分卡 / 草稿区
      查看每局明细（3 局） ›
```

- `onlineBar()` 去掉「邀请」「退出」两个按钮，只保留连接状态 + 房号 + 在玩/观战人数。
- 记分页正文删除那一排管理按钮。

### ⋯ 菜单内容（按身份）

| 视图 / 身份 | 菜单项 |
|---|---|
| 记分页 · 联机房主 | 玩家管理 / 改局权限开关（`✅ 牌友可改已存局` ↔ `🔒 改局仅房主`）/ 作废本场（danger）/ 结束本场（danger）/ 退出房间 |
| 记分页 · 联机非房主 | 退出房间 |
| 记分页 · 本地单机 | 玩家管理 / 作废本场（danger）/ 结束本场（danger） |
| **大厅（lobby）· 房主与牌友** | 退出房间 |

顶栏操作区：联机（含大厅）`[分享] [⋯]`；本地仅 `[⋯]`（本地无联机身份，无分享入口）。

**大厅必须一并处理**：`onlineBar()` 同时被大厅与记分页使用，把「邀请/退出」从它里面摘掉后，大厅会失去邀请与退出入口——而大厅恰恰是最需要邀请的地方（招呼人进来认领座位）。因此 `VIEWS.lobby` 的 `topbar` 也要传入 `[分享] [⋯]` 操作区，菜单项为「退出房间」，与现有能力保持等价（大厅原本也只有邀请/退出两项）。

**权限不变**：菜单只是收纳，各项仍沿用现有 `isOwner()` / `RunfastSync.canEdit()` 判断，不新增权限面。

## 3. 分享（优化点 2）

`App.invite()` 更名为 `App.share()`（记分页顶栏「分享」调用）：

1. 若 `navigator.share` 存在（安全上下文：HTTPS 或 localhost）→ `navigator.share({title, text, url})`；`AbortError` 静默返回，其它错误落到第 2 步。
2. 否则弹**自建分享面板**：大字房号、邀请链接、二维码 `<img src="/qr?text=<encodeURIComponent(link)>">`、「复制链接」按钮（复用现有 `copyToClipboard`，其 `execCommand` 降级在非安全上下文可用）。

牌友扫码即可直接进房，在局域网场景下比系统面板更实用。

### 服务器新增 `GET /qr`

`server.js` 增加路由：读 `?text=`，为空或超过 512 字符返回 400；否则复用已有 `qrSvg(text)` 返回 `image/svg+xml; charset=utf-8`，`Cache-Control: no-store`。

**为什么走服务端**：`src/vendor/qrcode.js` 有 56KB，塞进 dist 会让单文件从 60KB 近乎翻倍；而二维码只在局域网联机场景需要（GitHub Pages 版本无联机入口，分享面板根本不会出现）。

**不改动**：结算页「分享战绩图」保持现状（`share-card.js` 本就是原生优先 + 长按保存/下载降级）。

## 4. 每局明细独立页（优化点 3）

**泛化现有 `VIEWS.rounds`**，而非新写视图（内容完全相同，只有数据源、权限、返回目标不同）：

- **数据源**：若 `sessionCtx()` 存在且其 `id === view.sid`，用它（联机进行中的场不在 `db.sessions` 里）；否则按原逻辑查 `db.sessions`（结算/历史路径不变）。查不到则回首页。
- **可编辑**：`view.from === 'session'` 且（本地 或 `RunfastSync.canEdit(...)`）时，`roundRow` 显示改/删；结算/历史进入仍全只读。
- **返回**：`view.from === 'session'` → `App.goSession()`；否则维持 `App.goSettle(sid, from)`。

记分页正文底部改为一行入口「查看每局明细（N 局）›」；`N === 0` 时不显示入口，保留原空态提示。

### 编辑后的返回位置

从明细页点「改」进入记分录入页时，在 `view` 上带 `returnTo:'rounds'`；`saveRound()` 与 `cancelRecord()` 改为调用统一的 `afterRecord()`——`returnTo === 'rounds'` 则回明细页，否则回记分页。注意在导航前先捕获 `returnTo`（`go()` 会替换 `view`）。

## 5. 排行榜「我」标识（优化点 4）

比分卡每行：名字后追加 `<span class="me-tag">我</span>`，条件为 `online.active && myNames().has(p.name)`，其中 `myNames()` 由 `mySeatIdx()` 映射为名字集合。房主代占多座时多行都标（与草稿区现有「（我）」语义一致）。

仅记分页比分卡显示；草稿区现有的「（我）/（代填）」保持不变；结算页与历史页不加（无联机身份语义）。

---

## 6. 一并修复：`routeByPhase` 误踢子视图

**现状缺陷**：`routeByPhase()` 在 `phase === 'playing'` 时判断 `view.name !== 'session'` 就强制 `go({name:'session'})`。因此联机下任何房间更新（别人填一格、房主加人）都会把正在「玩家管理」的人弹回记分页；新的明细页同样会被弹出，等于不可用。

**修复**：定义 playing 阶段的合法子视图集合 `['session','record','players','rounds']`；当前视图属于该集合时只 `render()`，不属于才跳回 `session`。

`record` 视图的数据来自本地 `view.cards` 等，重绘不会丢失正在录入的内容。

## 7. 非目标（本次不做）

- 不改协作草稿的交互（赢家先定 → 各自填 → 确认 → 自动记局）
- 不改权限模型与服务器 `canWrite/canPatch`（仅新增只读的 `/qr`）
- 不改结算页、历史页布局
- 不改战绩图分享逻辑
- 不为局域网启用 HTTPS（自签证书会破坏「扫码即用」）

## 9. 风险与边界

- **面板与重绘竞态**：面板挂 `body` 不受 `render()` 影响；`go()` 内 `closeSheet()` 防止切页残留。
- **顶栏拥挤**：标题溢出省略，操作区 `flex-shrink:0`。
- **`/qr` 仅局域网可用**：Pages 单机版不存在联机入口，分享面板不会出现；`/qr` 只接受长度受限的 text，返回静态 SVG，无状态、无副作用。
- **明细页与结束本场**：联机中房主结束本场时，`onRoom` 走 `snapshotOnlineFinished` 跳结算页，会离开明细页——这是预期行为。
- **改局权限实时变化**：房主在菜单里切换「允许他人修改」后，非房主停留在明细页会随 `onRoom` 重绘，改/删按钮实时出现或消失。
